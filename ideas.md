<idea>
  **Design Movement**: **Scientific Brutalism / Lab Precision**
  **Core Principles**:
  1. **Raw Data First**: Données brutes et graphiques techniques comme éléments centraux, pas de décoration superflue.
  2. **Grid Precision**: Utilisation stricte de grilles visibles, rappelant les cahiers de laboratoire et les plans techniques.
  3. **High Contrast**: Contraste élevé pour une lisibilité maximale, inspiré des interfaces de machines industrielles.
  4. **Functional Aesthetics**: Chaque élément a une fonction, la beauté vient de l'utilité.

  **Color Philosophy**:
  - **Clinical White (#FFFFFF)** & **Deep Slate (#1E293B)**: Base stérile et professionnelle.
  - **Safety Orange (#F97316)**: Pour les alertes et les points critiques (inspiré des étiquettes de sécurité).
  - **Data Blue (#3B82F6)**: Pour les visualisations de données neutres.
  - *Intent*: Évoquer la rigueur, la sécurité et la précision scientifique.

  **Layout Paradigm**:
  - **Dashboard-centric**: Pas de "hero" traditionnel, mais une entrée directe dans les métriques clés.
  - **Split-screen**: Comparaison constante (Théorie vs Réalité, Attendu vs Mesuré).
  - **Modular Cards**: Conteneurs à bords carrés, bordures fines, ombres nulles.

  **Signature Elements**:
  - **Monospace Typography**: Pour les données chiffrées et les labels techniques.
  - **Visible Grid Lines**: Lignes de séparation fines et grises entre les sections.
  - **Technical Icons**: Icônes filaires (outline) très détaillées.

  **Interaction Philosophy**:
  - **Hover-to-Reveal**: Survoler un point de donnée affiche les détails techniques complets.
  - **Toggle Switches**: Basculer entre vues simplifiées et vues expertes.

  **Animation**:
  - **Instant Transitions**: Pas de fondus doux, changements d'état immédiats (comme un switch électrique).
  - **Data Loading**: Barres de progression linéaires précises.

  **Typography System**:
  - **Headings**: *JetBrains Mono* ou *Roboto Mono* (Technique, brut).
  - **Body**: *Inter* (Lisible, neutre) - *Note: Avoid Inter as main font per instructions, replacing with **Space Grotesk** for headings and **Public Sans** for body.*
</idea>

<idea>
  **Design Movement**: **Industrial Blueprint / Engineering Schematic**
  **Core Principles**:
  1. **Schematic Visuals**: Le site ressemble à un plan technique interactif (blueprint).
  2. **Structure over Decoration**: La structure est le design. Lignes de construction visibles.
  3. **Process Oriented**: Focus sur le flux séquentiel (Cycle : Vide -> Chauffage -> Stérilisation -> Séchage).
  4. **Instructional Clarity**: Typographie et iconographie de type "manuel d'instruction".

  **Color Philosophy**:
  - **Blueprint Blue (#1E3A8A)**: Fond sombre profond.
  - **Chalk White (#F8FAFC)**: Pour le texte principal.
  - **Warning Yellow (#FACC15)**: Pour les annotations et les points d'attention.
  - **Grid Cyan (#22D3EE)**: Pour les lignes de grille et les mesures.
  - *Intent*: Immerger l'utilisateur dans la mécanique interne de l'autoclave.

  **Layout Paradigm**:
  - **Diagrammatic**: Les sections sont reliées par des lignes de flux.
  - **Fixed Sidebar**: Navigation latérale ressemblant à une légende de plan.
  - **Horizontal Scroll**: Pour visualiser le cycle temporel (timeline) de gauche à droite.

  **Signature Elements**:
  - **Dashed Lines**: Pour les connexions secondaires.
  - **Technical Annotations**: Flèches et labels pointant vers les éléments clés.
  - **Isometric Illustrations**: Représentations 3D techniques de l'autoclave.

  **Interaction Philosophy**:
  - **Exploded View**: Cliquer sur une partie de l'autoclave pour voir ses composants (vue éclatée).
  - **Step-by-Step**: Navigation guidée étape par étape à travers le cycle.

  **Animation**:
  - **Draw-in**: Les lignes se dessinent comme tracées par un stylo technique.
  - **Mechanical Slide**: Les panneaux glissent mécaniquement.

  **Typography System**:
  - **Headings**: *Teko* ou *Rajdhani* (Industriel, carré).
  - **Body**: *Share Tech Mono* ou *Source Code Pro* (Technique).
</idea>
